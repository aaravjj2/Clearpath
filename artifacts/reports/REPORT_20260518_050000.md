## Before / After

**IMP-005:**
- Before: Single attempt per provider; any transient 429 or timeout propagated immediately
- After: Up to 3 retries with 1s, 2s, 4s (+jitter) backoff before giving up on a provider

**IMP-006:**
- Before: No rate limiting; a single client could overwhelm the server
- After: 429 response after 60 requests/minute per IP; configurable via env

**IMP-007:**
- Before: `02_upload_pdf.spec.ts` and `04_streaming_analysis.spec.ts` were missing from the e2e suite
- After: Both files exist with meaningful test cases covering upload zone, tab navigation, and streaming behavior

**IMP-009:**
- Before: Users could select a 100 MB file; the frontend would attempt the upload and wait for server rejection
- After: Client-side check immediately shows a friendly size-error message and aborts the upload
